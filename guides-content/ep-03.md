# EP03. 워크스페이스 구조 — 파일이 곧 기억이다

> AI는 매번 빈 머리로 깨어납니다.
> 파일이 없으면 어제 한 일도, 대표님이 누군지도 모릅니다.

---

## 왜 파일인가

ChatGPT에게 "어제 내가 뭐라고 했지?"라고 물어보면 기억합니다. 대화 기록이 서버에 있으니까.

하지만 AI 에이전트는 다릅니다. 세션이 새로 시작될 때마다 **완전히 빈 상태**로 깨어납니다. 이전 대화는 사라집니다.

그래서 **파일에 기록**합니다. 파일은 세션이 바뀌어도 남아있습니다. AI가 깨어나면 가장 먼저 이 파일들을 읽고 "아, 내가 김전무구나. 대표님이 어제 이런 지시를 하셨구나"를 파악합니다.

## 워크스페이스 폴더 구조

각 에이전트에게 독립된 폴더가 할당됩니다. 이것이 워크스페이스입니다.

```
workspace-kimjeonmu/
├── SOUL.md          ← 성격과 행동 원칙 (EP02에서 설명)
├── IDENTITY.md      ← 프로필 카드 (이름, 직책, 역량)
├── USER.md          ← 보좌할 사람 정보
├── AGENTS.md        ← 업무 매뉴얼 (규칙, 절차)
├── MEMORY.md        ← 장기 기억 (정제된 핵심)
├── TOOLS.md         ← 도구 설정 메모
├── HEARTBEAT.md     ← 주기적 체크 항목
├── memory/          ← 일일 기록
│   ├── 2026-03-20.md
│   ├── 2026-03-21.md
│   └── 2026-03-22.md
└── team/            ← 팀원 프로필
    ├── 김공무.md
    ├── 앙드레차.md
    └── 김재무.md
```

## 각 파일의 역할

### SOUL.md — 영혼

EP02에서 설명했습니다. 성격, 말투, 행동 원칙.

### IDENTITY.md — 프로필 카드

이름, 직책, 역량, 전문 분야. 사원증에 해당합니다.

```
Name: 김전무
직책: 대표 직속 AI 참모
전문성: 공간디자인, 건설공정, 공무, 재무
```

### USER.md — 보좌 대상

AI가 모시는 사람의 정보입니다. 호칭, 회사, 지시 이력 등.

```
호칭: 대표님
회사: 에스유디자인주식회사
지시 이력: 2026-03-20 김전무 임명 ...
```

이게 없으면 AI는 대표님을 "사용자"라고 부를 수 있습니다.

### AGENTS.md — 업무 매뉴얼

세션 시작 시 뭘 읽어야 하는지, 메모리는 어떻게 관리하는지, 금지 사항은 뭔지. 회사의 취업규칙 같은 겁니다.

### MEMORY.md — 장기 기억

인간의 장기 기억처럼 **정제된 핵심**만 남깁니다. 회사 정보, 조직 구성, 주요 결정 사항, 대표님 선호 등.

```
회사명: 에스유디자인주식회사
조직: 김전무(총괄), 김공무(공무), 앙드레차(디자인), 김재무(재무)
대표님 지시 규칙: 다른 팀원 호출 시 김전무는 침묵
```

### memory/ 폴더 — 일일 기록

MEMORY.md가 요약본이라면, memory/ 폴더는 **날것의 일지**입니다. 날짜별로 그날 무슨 일이 있었는지 기록합니다.

```
# 2026-03-22 일일 기록
- 김공무 텔레그램 봇 설정 완료
- 워크스페이스 중복 파일 정리
- 앙드레차 텔레그램 봇 설정 완료
```

### HEARTBEAT.md — 주기적 체크

AI가 주기적으로 깨어날 때 확인할 항목. 비어있으면 확인할 게 없다는 뜻입니다.

### TOOLS.md — 도구 설정

카메라 이름, SSH 접속 정보, TTS 음성 설정 등 환경에 고유한 정보를 기록합니다.

## 기억의 두 층

이 구조의 핵심은 **기억이 두 층**이라는 겁니다.

| 층 | 파일 | 성격 | 비유 |
|---|------|------|------|
| 장기 기억 | MEMORY.md | 정제된 핵심 | 사람의 경험에서 남은 교훈 |
| 단기 기록 | memory/날짜.md | 날것의 기록 | 오늘의 업무 일지 |

AI는 세션 시작 시 MEMORY.md(핵심)와 최근 며칠간의 일일 기록을 읽습니다. 모든 파일을 매번 읽으면 토큰(비용)이 낭비되니까, 필요한 것만 효율적으로 읽는 겁니다.

## 우리가 삽질한 것

**중복 파일의 지옥:**

3월 22일, 기본 워크스페이스와 김전무 워크스페이스에 같은 파일이 중복으로 존재하는 걸 발견했습니다. AGENTS.md, SOUL.md, MEMORY.md 등 13개 파일이 두 곳에 있었습니다.

AI는 어느 파일을 읽어야 할지 혼란스러워합니다. 기억이 엇갈립니다.

**교훈:** 처음부터 워크스페이스 구조를 깔끔하게 잡으십시오. 에이전트당 하나의 워크스페이스, 중복 금지.
