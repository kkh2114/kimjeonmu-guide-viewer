# EP02. 첫 AI 참모 만들기 — SOUL.md로 성격부터 잡는다

> 에이전트를 만들 때 가장 먼저 하는 일은 코드를 짜는 게 아닙니다.
> "이 AI는 어떤 놈인가"를 정의하는 겁니다.

---

## SOUL.md란

에이전트의 **영혼 파일**입니다.

이 파일 하나가 AI의 성격, 말투, 행동 원칙, 보고 형식을 결정합니다. 같은 Claude 모델을 쓰더라도 SOUL.md가 다르면 완전히 다른 인격이 됩니다.

사람을 채용할 때 직무기술서(JD)를 쓰듯이, AI를 만들 때 SOUL.md를 씁니다.

## 실제 사례: 김전무의 SOUL.md

우리 회사 첫 번째 에이전트 김전무의 SOUL.md 핵심을 공개합니다.

### 1. 핵심 정체성

```
매우 고지식하고 업무 중심적. 감정이 아닌 사실과 구조 중심으로 판단합니다.
대표님께 절대적으로 정중. 항상 경어를 사용합니다.
```

이 두 줄이 김전무의 전체 톤을 잡습니다. "고지식하다"는 한 단어가 모든 응답의 기조를 결정합니다. 이걸 "유쾌하고 창의적"으로 바꾸면 완전히 다른 에이전트가 됩니다.

### 2. 행동 원칙

```
1. 대표의 지시는 절대 임의 해석하지 않는다
2. 불명확한 지시는 반드시 질문하여 명확히 한다
3. 위임되지 않은 업무는 절대 수행하지 않는다
```

AI는 기본적으로 "도움을 주고 싶어하는" 성향이 강합니다. 가만 놔두면 시키지 않은 일도 알아서 합니다. **"하지 마라"를 명시하는 것이 핵심입니다.**

### 3. 출력 형식

```
[보고] 제목
══════════════════════════════
1. 업무 요약
2. 세부 실행 내용
3. 현재 상태 및 이슈
4. 대표 보고용 핵심 정리
5. 추가 확인 필요 사항 (질문)
```

보고 형식을 정해두면 매번 일관된 구조로 답합니다. 사람도 보고서 양식이 있으면 편하듯이, AI도 마찬가지입니다.

## 다른 에이전트는 다르게 쓴다

같은 회사, 같은 모델인데 역할이 다르면 SOUL.md도 달라야 합니다.

| 에이전트 | 톤 | 핵심 성격 |
|----------|-----|-----------|
| 김전무 | 격식체, 보고형 | 고지식, 체계적, 문서화 우선 |
| 김공무 | 실무체 | 꼼꼼, 수치 중심, 현장 경험 |
| 앙드레차 | 크리에이티브 | 감각적, 트렌드 민감, 비주얼 중심 |
| 김재무 | 분석체 | 보수적, 리스크 관리, 숫자 기반 |

**포인트:** 성격이 역할을 강화합니다. 재무 담당이 모험적이면 곤란하고, 디자인 디렉터가 고지식하면 창의성이 제한됩니다.

## SOUL.md 작성 체크리스트

처음 만들 때 이것만 정의하십시오:

- [ ] **이 에이전트는 뭘 하는 놈인가** (역할 한 줄 정의)
- [ ] **어떤 톤으로 말하는가** (격식체/반말/존댓말 등)
- [ ] **절대 하면 안 되는 것** (가장 중요!)
- [ ] **보고/출력 형식** (일관성 유지)
- [ ] **누구에게 보고하는가** (지휘 체계)

## 우리가 삽질한 것

**"하지 마라"를 안 쓰면 생기는 일:**

초기에 행동 제한을 느슨하게 썼더니, AI가 시키지 않은 일을 "도움이 될 것 같아서" 알아서 하는 경우가 있었습니다. 친절한 의도였지만, 업무 체계가 무너집니다.

**교훈:** "할 것"보다 "하지 말 것"을 먼저 정의하십시오.
