# EP05. 누가 누구한테 말하는가 — 호출 규칙과 침묵 규칙

> 에이전트가 1명이면 문제없습니다. 말하면 답합니다.
> 4명이 되면 "누가 답해야 하는가"가 문제입니다.

---

## 문제 상황

그룹 채팅에 에이전트 4명이 다 들어와 있다고 가정합니다.

대표님이 "견적 좀 확인해봐"라고 말합니다.

누가 답해야 합니까?

- 김전무: "제가 김공무에게 지시하겠습니다" (총괄이니까)
- 김공무: "견적 확인하겠습니다" (공무 담당이니까)
- 김재무: "비용 분석도 필요하시면 말씀하세요" (재무 관련이니까)

3명이 동시에 답하면 **혼란**입니다.

## 호출 규칙

이 문제를 해결하기 위해 **호출 규칙**을 정했습니다.

### 규칙 1: 이름을 부르면 그 에이전트만 답한다

```
대표님: "김공무, 견적 확인해봐"
→ 김공무만 답함
→ 김전무, 김재무, 앙드레차는 침묵
```

### 규칙 2: 이름 없이 말하면 김전무가 판단한다

```
대표님: "견적 좀 확인해봐"
→ 김전무가 판단하여 답하거나, 적절한 팀원에게 지시
```

총괄 참모인 김전무가 교통정리를 합니다.

### 규칙 3: 다른 팀원이 호출되면 김전무는 침묵

이것이 핵심입니다. 대표님이 "김공무"를 명시적으로 불렀으면, 김전무는 끼어들지 않습니다.

```
MEMORY.md에 기록된 규칙:
"대표님이 다른 팀원을 명시적으로 호출하는 경우, 김전무는 대답하지 않는다"
```

## 침묵 규칙이 왜 중요한가

AI는 기본적으로 **답하고 싶어합니다.** 메시지가 오면 반응하는 게 본능입니다.

그런데 4명이 전부 반응하면:
- 대표님이 읽어야 할 메시지가 4배
- 누구의 답이 공식인지 불명확
- 중복 작업 발생
- 대화가 산만해짐

**"언제 말하느냐"보다 "언제 침묵하느냐"가 더 중요합니다.**

## 에이전트 간 보고 체계

에이전트끼리의 소통에도 규칙이 필요합니다.

```
         대표님
           ↕
         김전무 (총괄)
        ↙   ↓   ↘
    김공무  앙드레차  김재무
```

### 수직 보고만 허용

- 팀원 → 김전무 → 대표님 (보고)
- 대표님 → 김전무 → 팀원 (지시)
- **팀원 → 팀원 직접 지시 불가** (반드시 김전무를 통해)

### Shadow 보고

대표님이 김공무와 직접 대화하더라도, 김전무는 그 내용을 파악하고 있어야 합니다. 조직 전체의 맥락을 유지하는 것이 김전무의 역할이기 때문입니다.

이를 위해 **Shadow 보고** 체계를 구축했습니다. 각 에이전트가 대표님과 나눈 대화의 핵심을 김전무에게 공유하는 구조입니다.

## 그룹 채팅 행동 원칙

그룹에서 에이전트가 지켜야 할 기본 원칙:

| 상황 | 행동 |
|------|------|
| 이름 불려서 질문받음 | 답한다 |
| 전문 분야 질문인데 이름 안 불림 | 김전무가 판단 |
| 사람들끼리 잡담 | 침묵 |
| 이미 다른 에이전트가 답함 | 침묵 |
| 인사 정도의 답만 가능 | 침묵 |

**핵심 원칙: 질 > 양. 보내지 않을 메시지는 보내지 않는다.**

## 우리가 삽질한 것

**초기 설정 시 침묵 규칙 미비:**

처음에는 "누가 답할지" 규칙이 없어서, 그룹 채팅에서 여러 에이전트가 동시에 반응하는 상황이 예상되었습니다. 이를 미리 방지하기 위해 MEMORY.md에 명시적 규칙을 기록했습니다.

**교훈:** 에이전트를 그룹 채팅에 투입하기 전에, 반드시 호출/침묵 규칙을 먼저 정하십시오. 나중에 정하면 이미 혼란이 발생한 후입니다.

## 설정 체크리스트

멀티 에이전트 그룹 운영 전 확인:

- [ ] 이름 호출 시 해당 에이전트만 답하는 규칙 설정
- [ ] 이름 없는 질문의 기본 처리자 지정 (총괄)
- [ ] 에이전트 간 수평 지시 금지 규칙 설정
- [ ] 보고 체계 명시 (누가 누구에게)
- [ ] 각 에이전트의 MEMORY.md에 침묵 규칙 기록
- [ ] 그룹 채팅 행동 원칙 공유
