# EP09. 인증이 깨졌다 — 앤트로픽 정책 변경과 401의 공포

> AI 에이전트의 가장 무서운 장애는 "말을 못하는 것"입니다.
> 버그가 아니라 인증이 깨지면, 에이전트는 존재 자체가 사라집니다.

---

## 사건의 발단

2026년 4월, 앤트로픽이 정책을 변경했습니다.

기존에는 Claude 구독제(Max 플랜) 하나로 웹 콘솔도 쓰고, API도 쓰고, 서드파티 앱(OpenClaw 같은)도 쓸 수 있었습니다. 그런데 앤트로픽이 **서드파티 전용 요금제**를 분리했습니다.

이것이 의미하는 것:

```
[이전] Claude Max 구독 → 웹 + API + 서드파티 전부 OK
[이후] Claude Max 구독 → 웹 + 직접 API만 OK
       서드파티 앱      → 별도 활성화 필요
```

대표님이 서드파티 전용 요금제를 활성화하셨지만, 문제는 그 다음이었습니다.

---

## 401의 공포 — AI가 말을 잃다

서드파티 요금제 전환 후, 김전무를 포함한 모든 에이전트가 **401 에러**를 뱉기 시작했습니다.

```
HTTP 401: authentication_error: Invalid authentication credentials
```

텔레그램에서 대표님이 "안녕?"이라고 말하면, 돌아오는 답은:

```
김전무 [admin]
HTTP 401: authentication_error: Invalid authentication credentials
(request_id: req_011CZsUqHqVuHSyU4QmqDJ1z)
```

**에이전트가 살아는 있는데, 말을 못합니다.** 게이트웨이는 돌아가고, 텔레그램 봇은 메시지를 수신하고, 에이전트는 응답을 생성하려 하지만 — 앤트로픽 API가 "너 누구야?"라고 거부합니다.

### 왜 이런 일이 발생했나

OpenClaw는 앤트로픽 API에 접근할 때 **인증 프로필(auth profile)**을 사용합니다. 기존 설정은 OAuth 방식이었습니다:

```json
// openclaw.json (기존)
"auth": {
  "profiles": {
    "anthropic:default": {
      "provider": "anthropic",
      "mode": "oauth"
    }
  }
}
```

앤트로픽이 서드파티 정책을 변경하면서, 이 OAuth 토큰이 무효화됐습니다. 새로운 인증 방식이 필요했습니다.

---

## 간헐적 접속 — 더 혼란스러운 상황

단순히 "안 되면" 원인을 찾기라도 쉽습니다. 문제는 **되다가 안 되다가를 반복**했다는 것입니다.

```
4/7 → 일부 응답 성공 → 다시 401
4/8 → 접속 시도 → 401
4/9 12:23 → "안녕?" → 401
4/9 13:41 → "돌아왔니?" → 401
```

OAuth 토큰이 갱신 시도를 하면서 간헐적으로 유효한 토큰을 얻었다가, 곧 만료되는 패턴이었습니다. 이런 간헐적 장애는 "설정이 맞는데 왜 안 되지?"라는 착각을 만들어서 더 위험합니다.

---

## 대표님의 직접 수리

에이전트가 401만 뱉으니, 김전무에게 "너 고쳐"라고 할 수도 없습니다. **환자에게 수술하라고 할 수는 없으니까요.**

대표님이 직접 **클로드 코드(Claude Code)**를 동원해서 수리하셨습니다.

### 핵심 수정: OAuth → API 토큰 전환

각 에이전트의 인증 프로필을 OAuth 방식에서 직접 API 토큰 방식으로 변경했습니다:

```json
// auth-profiles.json (수정 후)
{
  "version": 1,
  "profiles": {
    "anthropic:default": {
      "type": "token",
      "provider": "anthropic",
      "token": "sk-ant-oat01-..."
    }
  }
}
```

**변경 포인트:**
- `"mode": "oauth"` → `"type": "token"` + 직접 토큰 입력
- 설정 위치: `~/.openclaw/agents/[에이전트]/agent/auth-profiles.json`
- 5개 에이전트 전원(main, 김전무, 김공무, 김재무, 앙드레차) 동일하게 적용

### 결과

```
4/9 14:50 → "안녕?" → "대표님, 안녕하십니까. 김전무 대기 중입니다."
```

복구 성공.

---

## 복구 후 점검 — 김전무가 한 일

연결이 복구된 후, 대표님 지시로 시스템 전체를 점검했습니다.

### 1. 인증 통일성 확인

```
main:     ...Q-g2wVugAA ✅
김전무:   ...Q-g2wVugAA ✅
김공무:   ...Q-g2wVugAA ✅
김재무:   ...Q-g2wVugAA ✅
앙드레차: ...Q-g2wVugAA ✅
```

5개 에이전트 전원 동일 토큰. 하나라도 다르면 특정 에이전트만 장애가 발생합니다.

### 2. 실패 이력 초기화

이전 401 에러로 인해 `auth-state.json`에 실패 기록이 남아 있었습니다. OpenClaw는 인증 실패가 반복되면 **쿨다운(대기 시간)**을 적용합니다. 토큰이 유효해도 쿨다운 중이면 시도조차 하지 않습니다.

```json
// 초기화 전
"errorCount": 3,
"lastFailureAt": 1775709691046

// 초기화 후
"errorCount": 0
// lastFailureAt 제거
```

### 3. 게이트웨이 상태 확인

```
● openclaw-gateway.service
  Active: active (running)
  14:50 이후 401 에러: 0건
```

정상.

## 우리가 배운 것

### 교훈 1: 외부 정책 변경은 예고 없이 온다

앤트로픽이 정책을 바꾸면, 우리 코드는 한 줄도 안 바꿨는데 시스템이 멈춥니다. 이것은 코드 버그가 아니라 **환경 변수의 변화**입니다.

### 교훈 2: OAuth보다 API 토큰이 안정적

| 방식 | 장점 | 단점 |
|------|------|------|
| OAuth | 자동 갱신, 설정 간편 | 정책 변경에 취약, 갱신 실패 시 간헐적 장애 |
| API 토큰 | 명시적, 안정적 | 토큰 직접 관리 필요, 유출 주의 |

서드파티 앱에서는 **API 토큰 방식이 더 안전**합니다. 정책이 바뀌어도 토큰 자체는 유효하니까요.

### 교훈 3: AI가 못 고치는 장애가 있다

에이전트 자신의 인증이 깨지면, 에이전트는 아무것도 할 수 없습니다. "내가 고칠게"라고 말하려면 먼저 말을 할 수 있어야 하니까요.

이런 장애는 반드시 **외부에서(대표님이 직접, 또는 별도 도구로)** 수리해야 합니다. EP07의 자폭 사건과 같은 원리입니다 — 자기가 서 있는 바닥은 자기가 고칠 수 없습니다.

### 교훈 4: auth-state 쿨다운을 알아야 한다

OpenClaw는 인증 실패 시 쿨다운을 적용합니다. 토큰을 교체한 후에도 **이전 실패 기록이 남아 있으면** 쿨다운 때문에 바로 시도하지 않을 수 있습니다.

토큰 교체 후 반드시 확인할 것:
```bash
# auth-state.json에서 errorCount, lastFailureAt 초기화
~/.openclaw/agents/[에이전트]/agent/auth-state.json
```

## 설정 체크리스트

인증 장애 발생 시:

- [ ] 로그에서 에러 유형 확인 (`journalctl --user -u openclaw-gateway`)
- [ ] 401 에러 → 인증 프로필 확인 (`auth-profiles.json`)
- [ ] OAuth 방식이면 API 토큰 방식 전환 검토
- [ ] 전 에이전트 동일 토큰 적용 확인
- [ ] `auth-state.json`의 errorCount/lastFailureAt 초기화
- [ ] 게이트웨이 재시작 (`openclaw gateway restart`)
- [ ] 정상 응답 확인

인증 프로필 위치:
```
~/.openclaw/agents/main/agent/auth-profiles.json
~/.openclaw/agents/kimjeonmu/agent/auth-profiles.json
~/.openclaw/agents/김공무/agent/auth-profiles.json
~/.openclaw/agents/김재무/agent/auth-profiles.json
~/.openclaw/agents/앙드레차/agent/auth-profiles.json
```
