# EP04. 텔레그램에 봇 연결하기 — 멀티 계정의 세계

> AI에게 영혼(SOUL.md)을 넣고, 방(워크스페이스)을 줬습니다.
> 이제 바깥 세상과 소통할 통로를 뚫어야 합니다.

---

## 구조를 먼저 이해하자

AI 에이전트가 텔레그램으로 메시지를 주고받으려면 세 가지가 필요합니다.

```
텔레그램 봇 ←→ OpenClaw 게이트웨이 ←→ AI 에이전트
```

1. **텔레그램 봇**: @BotFather로 만듭니다. 텔레그램에서의 "얼굴"입니다.
2. **OpenClaw 게이트웨이**: 봇이 받은 메시지를 AI에게 전달하는 중간 다리.
3. **AI 에이전트**: 실제로 생각하고 답하는 주체.

에이전트가 4명이면, 봇도 4개, 연결도 4개입니다.

## Step 1: 텔레그램 봇 만들기

텔레그램에서 @BotFather를 검색하고 대화를 시작합니다.

```
/newbot
→ 봇 이름 입력 (예: 에스유 김공무)
→ 봇 username 입력 (예: sudesign_kimgongmu_bot)
→ 토큰 발급됨 (이걸 OpenClaw에 등록)
```

이걸 에이전트 수만큼 반복합니다.

| 에이전트 | 봇 이름 | 용도 |
|----------|---------|------|
| 김전무 | 기존 default 봇 | 총괄 참모 |
| 김공무 | kimgongmu 봇 | 공무 담당 |
| 앙드레차 | andrecha 봇 | 디자인 담당 |
| 김재무 | kimjaemu 봇 | 재무 담당 |

## Step 2: OpenClaw 텔레그램 계정 등록

OpenClaw 설정 파일에 텔레그램 계정을 추가합니다.

기본 구조는 "default" 계정 하나지만, 여러 봇을 운영하려면 **멀티 계정**으로 설정해야 합니다.

```yaml
# 설정 파일 구조 (개념)
텔레그램:
  default:     → 김전무 봇 토큰
  kimgongmu:   → 김공무 봇 토큰
  andrecha:    → 앙드레차 봇 토큰
  kimjaemu:    → 김재무 봇 토큰
```

각 계정 이름(kimgongmu, andrecha 등)은 자유롭게 정하면 됩니다.

## Step 3: 바인딩 — 봇과 에이전트 연결

계정을 만들었으면, "이 봇으로 온 메시지는 이 에이전트가 처리한다"는 연결을 해야 합니다. 이것이 **바인딩**입니다.

```
default 계정  →  kimjeonmu 에이전트
kimgongmu 계정  →  kimgongmu 에이전트
andrecha 계정  →  andrecha 에이전트
kimjaemu 계정  →  kimjaemu 에이전트
```

바인딩이 없으면 봇이 메시지를 받아도 어느 에이전트한테 전달할지 모릅니다.

## Step 4: 게이트웨이 재시작

설정을 변경한 후에는 **게이트웨이 재시작**이 필요합니다. 재시작 없이는 새 설정이 반영되지 않습니다.

```
openclaw gateway restart
```

## 멀티 에이전트의 장점

봇이 분리되어 있으면:

- 대표님이 **김공무에게** 견적 질문하면, 김공무 봇으로 메시지를 보냅니다
- **김전무에게** 전체 현황을 물으면, 김전무 봇으로 메시지를 보냅니다
- 각 봇이 독립적이니 대화가 섞이지 않습니다
- 그룹 채팅에 여러 봇을 초대할 수도 있습니다

## 우리가 삽질한 것

**게이트웨이 재시작 문제:**

3월 22일, 모든 봇 설정을 완료했지만 OpenClaw 버전 문제로 게이트웨이 재시작이 안 됐습니다. 설정은 했는데 반영이 안 되는 상황.

원인: 버전 업데이트에 sudo 권한이 필요했고, 업데이트 없이 재시작하면 호환성 문제가 발생할 수 있었습니다.

**교훈:** 봇 추가 전에 OpenClaw 버전이 최신인지 먼저 확인하십시오. 버전 업데이트 → 봇 설정 → 게이트웨이 재시작 순서를 지키십시오.

**워크스페이스 보조 파일 누락:**

새 에이전트를 만들고 봇은 연결했는데, USER.md, MEMORY.md 같은 보조 파일을 빼먹었습니다. 에이전트가 깨어났을 때 "대표님이 누구인지" 모르는 상황이 발생합니다.

**교훈:** 에이전트 생성 체크리스트를 만들어두십시오.

```
□ SOUL.md (성격)
□ IDENTITY.md (프로필)
□ USER.md (보좌 대상)
□ AGENTS.md (업무 규칙)
□ MEMORY.md (장기 기억)
□ TOOLS.md (도구 설정)
□ HEARTBEAT.md (주기 체크)
□ memory/ 폴더 생성
□ 텔레그램 봇 연결
□ 바인딩 설정
```
