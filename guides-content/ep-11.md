# EP11. 컨텍스트 다이어트 — 토큰을 절반으로 줄이다

> 토큰은 돈입니다. 안 읽어도 되는 파일을 매번 읽으면 그건 매 세션마다 동전을 버리는 겁니다.

---

## 발단 — "왜 이렇게 토큰이 많이 먹지?"

평범한 대화였습니다. Tailscale 접속 문제 확인하고, GitHub 푸시 충돌 해결하고. 복잡한 작업도 아닌데 토큰 소모가 눈에 띄게 많았습니다.

대표님이 물으셨습니다. "이런 평범한 대화에도 왜 이렇게 토큰이 많이 먹는 거지?"

좋은 질문이었습니다. 답을 찾기 위해 워크스페이스를 뜯어봤습니다.

---

## 진단 — 세션마다 38KB가 기본으로 올라간다

Claude Code는 워크스페이스 루트의 마크다운 파일과 memory 디렉토리를 **매 세션 시작 시 전부 로드**합니다. 파일이 많을수록, 길수록, 매번 더 많은 토큰을 소모합니다.

### 루트 마크다운 (9개 파일, 13.9KB)

| 파일 | 줄 수 | 역할 | 판정 |
|------|-------|------|------|
| AGENTS.md | 71줄 | 운영 규칙 | 유지 |
| COOPERATION.md | 167줄 | 협조 체계 | **축약 대상** |
| SOUL.md | 65줄 | 정체성 | 유지 |
| MEMORY.md | 44줄 | 장기 기억 인덱스 | 유지 |
| TOOLS.md | 19줄 | 환경 설정 | 유지 |
| ORG_CHART.md | 71줄 | 조직도 | **중복 → 삭제** |
| USER.md | 17줄 | 사용자 정보 | **빈 템플릿 → 삭제** |
| IDENTITY.md | 23줄 | 정체성 템플릿 | **SOUL.md와 중복 → 삭제** |
| HEARTBEAT.md | 5줄 | 하트비트 설정 | **AGENTS.md에 흡수** |

### memory 디렉토리 (14개 파일, 24.3KB)

3월 20일부터 4월 14일까지 14개의 일일 업무일지가 전부 로드되고 있었습니다. 3주 전 기록까지 매번 읽을 필요는 없습니다.

### 데모 파일 (12KB)

시연이 끝난 `demo-briefing.md`, `demo-scenario-v2.md`, `demo-data/`가 여전히 루트에 있었습니다.

**합계: ~38KB (~9,500토큰)가 매 세션 기본 소모.**

---

## 수술 — 4단계 정리

### 1단계: COOPERATION.md 축약 (167줄 → 39줄)

가장 큰 단일 파일이었습니다. ASCII 도표와 반복 설명을 제거하고 핵심 규칙만 남겼습니다.

**변경 전:**
```
┌─────────────────────────────────────┐
│              대표님                  │
│  (모든 에이전트 직접 지시·보고 가능) │
└──────────┬──────────────────────────┘
           │ 정식 라인
           ▼
┌─────────────────────────────────────┐
│   🎖️ 김전무 (총괄 참모 / 수장)      │
...
```

**변경 후:**
```
지휘 구조: 대표님 → 김전무(총괄참모/수장) → 김공무(현장실무), 앙드레차(크리에이티브) / 김공무 → 김재무(재무통제)
```

8줄이 1줄로. AI는 ASCII 박스가 없어도 구조를 동일하게 이해합니다. 도표는 사람용 장식이지 AI에게는 토큰 낭비입니다.

**기능 영향: 없음.** 지휘 구조, Shadow 보고, 견제 관계, 에스컬레이션 규칙 전부 그대로 유지.

### 2단계: 중복 파일 삭제

- **ORG_CHART.md (71줄):** COOPERATION.md + MEMORY.md에 이미 같은 정보가 있었습니다. 삭제.
- **USER.md (17줄):** 거의 빈 템플릿. 실제 정보는 SOUL.md에 있었습니다. 삭제.
- **IDENTITY.md (23줄):** 역시 빈 템플릿. SOUL.md가 정체성을 담당합니다. 삭제.
- **HEARTBEAT.md (5줄):** 3줄짜리 내용을 AGENTS.md 하트비트 섹션에 합쳤습니다. 삭제.

### 3단계: 데모 파일 삭제

시연 세션은 이미 끝났습니다. `demo-briefing.md`, `demo-scenario-v2.md`, `demo-data/` 전부 삭제. 12KB 절약.

### 4단계: 오래된 memory 아카이빙

3월 20일~31일까지의 업무일지 8개를 `memory/archive/`로 이동. 최근 6개(4월분)만 활성 상태로 유지. 14.5KB 절약.

| 항목 | 변경 전 | 변경 후 | 절약 |

| 루트 MD 파일 | 9개, 13.9KB | 5개, 9.6KB | 4.3KB |

하루 10세션 기준, 일일 ~55,000토큰 절약. 월간 약 165만 토큰.

## 핵심 원칙

### 1. AI에게 ASCII 도표는 장식이다

사람은 박스와 화살표로 구조를 파악합니다. AI는 "A → B → C"라는 텍스트만으로 동일하게 이해합니다. 시각적 표현에 쓰이는 특수문자(┌─┐│▼└)는 전부 토큰 낭비입니다.

### 2. 중복은 비용이다

같은 정보가 ORG_CHART.md, COOPERATION.md, MEMORY.md에 세 번 들어있으면, 세 번 읽고, 세 번 토큰을 씁니다. 하나만 남기고 나머지는 지워야 합니다.

### 3. 오래된 기억은 아카이빙한다

3주 전 업무일지를 매 세션마다 읽을 이유가 없습니다. 필요하면 아카이브에서 꺼내 보면 됩니다. 활성 메모리는 최근 며칠분만 유지합니다.

### 4. 안 쓰는 파일은 바로 치운다

데모가 끝났으면 데모 파일을 지웁니다. "나중에 또 쓸지 모르니까"라고 남겨두면, 매 세션마다 12KB씩 토큰이 새어나갑니다.
